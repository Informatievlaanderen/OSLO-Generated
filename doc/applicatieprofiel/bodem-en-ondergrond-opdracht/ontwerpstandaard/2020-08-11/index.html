Afhankelijkheden

Dit document beschrijft een applicatieprofiel, in dit geval Opdracht (Applicatieprofiel). Dit applicatieprofiel beantwoordt de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

Entiteiten

Bijlage

Beschrijving
Definitie van bijlagen.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: auteurs, bestand, bijlage type, datum, jaar, opmerking, titel, url.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
auteurs String 0..1 Naam of namen van de auteur(s) van de bijlage.
bestand String 1 Naam van het bestand dat als bijlage wordt gebruikt.
bijlage type Bijlage Type 0..1 Het type document van de bijlage.
datum Date 1 Datum van de bijlage.
jaar Integer 1 Jaar van de bijlage. Een getal van 4 cijfers.
opmerking Opmerking 0..1 Opmerking bij de bijlage.
titel String 1 Titel/naam van de bijlage.
url URI 1 URL waarnaar de bijlage verwijst.

Opdracht

Beschrijving
Opdracht.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: bijlage, dataleverancier, einddatum, kwaliteit, locatie, naam, omschrijving, opdrachtgever, opdrachtnemer, opmerking, startdatum, status.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
bijlage Bijlage 0..* Bijlage bij de Opdracht.
dataleverancier Dataleverancier 0..1 De organisatie die de data aangeleverd heeft.
einddatum Date 0..1 Datum dat de opdracht afgesloten werd.
kwaliteit Kwaliteit Opdracht 0..1 TO DO
locatie Geometrie 0..1 Een veelhoek die de locatie van de opdracht aangeeft. Een veelhoek wordt opgegeven via WKT syntax.
naam String 1 Unieke identificatie van de opdracht.
omschrijving String 0..1 Korte beschrijving van de inhoud van de opdracht.
opdrachtgever Opdrachtgever 1 De opdrachtgever.
opdrachtnemer Opdrachtnemer 1 De opdrachtnemer.
opmerking Opmerking 0..* TO DO
startdatum Date 0..1 Datum dat de opdracht werd opgestart.
status Status Type 0..1 Status van de opdracht.

Opmerking

Beschrijving
Bijkomende informatie.
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: auteur, datum, tekst.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
auteur Persoon 0..1 De persoon van wie de opmerking afkomstig is.
datum Date 0..1 Datum van de opmerking.
tekst String 1 Tekst van de opmerking.

Datatypes

Geometrie

Beschrijving
Vorm- en positiekenmerken van een object.
Eigenschappen
Voor dit datatype zijn de volgende eigenschappen gedefinieerd: coordinatenstelsel, wkt.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
coordinatenstelsel 1 De EPSG code van het coordinatenstelsel.
wkt 1 De geometrie als WKT.

Kwaliteit Opdracht

Beschrijving
Gegevens in verband met de kwaliteit van de opdracht.
Eigenschappen
Voor dit datatype zijn de volgende eigenschappen gedefinieerd: aard, controle, origine.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
aard Opdracht Aard 0..1 Geeft een indicatie van de thematiek waarbinnen de gegevens verzameld worden binnen de opdracht
controle String 0..1 TO DO
origine Opdracht Origine 0..1 Geeft een indicatie van de herkomst van het ontstaan van de gegevens binnen de opdracht.
