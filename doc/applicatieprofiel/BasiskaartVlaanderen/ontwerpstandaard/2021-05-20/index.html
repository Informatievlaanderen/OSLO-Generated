Afhankelijkheden

Dit document beschrijft een applicatieprofiel, in dit geval Basiskaart Vlaanderen Applicatieprofiel. Dit applicatieprofiel beantwoordt de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

Administratief Perceel

Beschrijving
Een administratief perceel (Adp) is een grafische voorstelling van de vermoedelijke ligging van het kadastraal perceel, waarbij de perceelsgrenzen werden geïnterpreteerd rekening houdend met de terreinsituatie.

Eigenschappen

isToegekendAan Adres 0..* Adresseerbaar object waaraan het adres is toegekend.
labelBeheerder String 1 Een beknopte beschrijving van de organisatie die verantwoordelijk is voor het beheer van de perceelsinformatie.

De watergang beslaat het gebied dat rechtstreeks gedomineerd wordt door de fysieke aanwezigheid van oppervlaktewater (waterlopen en stilstaande wateroppervlakken).
Subklasse van
Waterdeel, Deel
Eigenschappen
Voor deze entiteit zijn de volgende eigenschappen gedefinieerd: isGerelateerdAan, lengteOmtreklijn, naam, opnamedatum.
Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
isGerelateerdAan Waterloopsegment 1..* Duidt het waterloopsegment aan waaraan de watergang kan gekoppeld worden.
lengteOmtreklijn Kwantitatieve Waarde 1 Lengte in meter van de geometrie van de omtreklijn.
naam String 1 Naam die de waterloop draagt bij zijn monding. Dit is de zogenaamde VHA-roepnaam van de waterloop maar kan ook de waarden nvt (niet van toepassing) of ng (niet gekend) aannemen.
opnamedatum DateTime 1 Datum van oplevering.

website URI 0..1 Website waarnaar men kan surfen.
